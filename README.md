# Real-Time Fraud Detection System

## 🧠 Overview

A real-time fraud detection system that analyzes events and assigns risk scores to transactions.

---

## 🎯 Problem Statement

Fraud detection requires:

* Low latency decisions
* Real-time data processing
* Adaptive models

---

## 🏗️ Architecture

* Event ingestion
* Feature store
* Scoring service
* Decision engine

---

## ⚙️ Features

* Real-time scoring (<100ms)
* Rule-based + ML-based detection
* Feedback loop (learning)
* Risk classification

---

## 🛠️ Tech Stack

* Python (ML / scoring)
* Node.js / NestJS
* Kafka / SQS
* Redis (feature store)
* PostgreSQL

---

## 🔥 Challenges

* Latency constraints
* False positives
* Feature consistency
* Scaling under load

---

## 📊 Metrics

* Fraud detection rate
* False positives
* Latency (p95)
* Throughput

---

## 🧪 Failure Scenarios

* Missing features
* Delayed events
* Model drift

---

* [ ] Define scoring model

---

## 💬 Pitch

Built a real-time fraud detection system capable of scoring transactions under 100ms, reducing fraud and improving decision accuracy.
